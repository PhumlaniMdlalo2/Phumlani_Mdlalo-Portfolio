## Reflection 1: Coding My CV in Markdown

**Situation**

I was required to write my CV using Markdown language and deploy it on GitHub Pages. I had never written in Markdown before and was unfamiliar with its syntax, so this was a completely new skill I had to learn from scratch.

**Task**

My goal was to produce a well-structured, professionally formatted CV using GitHub's built-in editor, making use of Markdown features such as headings, tables, and links to present my personal details, education, experience, skills, and projects clearly.

**Action**

I used the GitHub editor to write my CV directly in the browser, which was helpful because it offered a live preview so I could see how the Markdown rendered in real time. I taught myself the syntax by experimenting with different elements as I built each section. The most challenging parts were creating tables for the contact section and ensuring the formatting remained consistent throughout. I also ran into a significant challenge when trying to embed my mock interview video — the file was larger than 60MB, which meant I could not simply upload it to the repository and link it through standard Markdown. I had to find a workaround to make the video accessible via GitHub Pages.

**Result**

By the end of the task, I had successfully written my full CV in Markdown and published it to GitHub Pages. More importantly, I gained a working understanding of Markdown syntax that I did not have before — a skill that is directly relevant to professional software development, where Markdown is used daily in README files, pull requests, and documentation. The challenge with the tables pushed me to be more careful and methodical, and working through the video file size issue taught me to problem-solve under constraints rather than give up.

## Reflection 2: Mock Interview Experience

**Situation**

I was required to record a mock interview video of no longer than two minutes, marketing myself to a potential employer in the ICT field. The video needed to demonstrate interview skills covering personality, abilities, qualifications, and behavioural questions.

**Task**

My task was to prepare and record a professional self-introduction video that would simulate a real job interview scenario. I needed to dress appropriately, choose a plain background, and deliver my answers confidently and clearly, as if speaking directly to a hiring manager.

**Action**

Before recording, I reviewed the seven types of interview questions in the subject guide and prepared what I wanted to say about my background, skills, and career goals. I dressed in interview-appropriate attire and set up a clean background for filming. Going into the recording, I felt nervous — speaking directly to a camera without another person there to respond felt unnatural and uncomfortable. I pushed through that discomfort and completed the recording, focusing on getting my key points across within the two-minute limit.

**Result**

After finishing the video, I noticed a significant shift in how I felt. The nervousness I had going in had reduced, and I felt more comfortable with the process of expressing myself while being recorded. Watching it back, however, I recognised areas for improvement — specifically, my tone was too flat and my energy level was lower than it would need to be to make a strong impression on a real employer. If I were to record it again, I would be more deliberate about varying my tone and projecting more enthusiasm. This exercise taught me that interview confidence is a skill that requires practice, and that the discomfort of doing it once makes the next time significantly easier.

## Reflection 3: Publishing on GitHub Pages

**Situation**

As part of the GitHub Portfolio module in PRP37XS, I was required to publish my CV and mock interview video to a live website using GitHub Pages. This was the first time I had ever used GitHub Pages or deployed any website to the internet.

**Task**

My goal was to create a public-facing portfolio site hosted at my GitHub Pages URL, containing my Markdown CV and embedded mock interview video, so that it could serve as a professional digital portfolio accessible to potential employers.

**Action**

I followed the steps outlined in the subject guide — creating a repository named in the `username.github.io` format, enabling GitHub Pages in the repository settings, and deploying from the main branch. The process was not without difficulty. I encountered broken links where certain content was not rendering correctly after deployment, and there was a delay between pushing changes to the repository and seeing them go live on the site, which made it harder to quickly identify and fix issues. I also had to troubleshoot the deployment settings more than once before the site published correctly.

**Result**

Despite the challenges, I successfully deployed my portfolio to GitHub Pages and had a live, publicly accessible site by the end of the process. The most valuable thing I took from this experience was understanding how deployment works in practice — that there is a difference between code that works locally and content that renders correctly once published. Having my portfolio live online is also practically useful, as it means I can include a URL in job applications and give potential employers direct access to my work. This experience gave me foundational knowledge of web deployment that connects directly to the kind of technical skills expected in the ICT industry.
